.About this task

* Changing the protocol on the host I/O modules involves preparing for the protocol change, stopping the host I/O operations, changing the protocol, and then configuring the host to use the new protocol.

* The protocol is changed on all host I/O modules in your storage system.

* Changing the protocol on the host I/O modules temporarily disrupts host access to all volumes and causes the storage system to be unavailable during the reboot process (both controllers are rebooted), which can take a few minutes.

//* New storage system installations and new controllers being used in a controller upgrade ship with a default host protocol. You need to check if the default host protocol is what you want and if not, you need to change it.


== Step 1: Prepare for the protocol change

Prepare to change the protocol on the host I/O modules in your storage system by verifying the current protocol on the host I/O modules, determining if you need to change the protocol, and if yes, scheduling a downtime maintenance window, and obtaining any necessary hardware.

.Steps

. If you do not know the current protocol on the host I/O modules in your storage system, verify it using SANtricity System Manager.

.. Select *Settings* > *System*.
.. In the *Additional Settings* section, click *Change host I/O protocol* to open that page.
.. See the default protocol used on the host I/O modules in your storage system displayed in the *Host I/O Protocol* field.

== Step 2: Stop host I/O operations

You must stop all I/O operations from the host before changing the protocol on the host I/O modules.

NOTE: You cannot access data on the storage system until you successfully complete the protocol change.

//Need to create includes
//syntax example:
// include::../_include/ge_io_module_upgrade_power_off_controllers.adoc[]

.Steps

. Ensure that no I/O operations are occurring between the storage system and all connected hosts. For example, you can perform these steps:
 ** Stop all processes that involve the LUNs mapped from the storage to the hosts.
 ** Ensure that no applications are writing data to any LUNs mapped from the storage to the hosts.
 ** Unmount all file systems associated with volumes on the storage system.
+
NOTE: The exact steps to stop host I/O operations depend on the host operating system and the configuration, which are beyond the scope of these instructions. If you are not sure how to stop host I/O operations in your environment, consider shutting down the host or contact https://mysupport.netapp.com/site/global/dashboard[NetApp Support^].
+
CAUTION: *Possible data loss* -- If you continue this procedure while I/O operations are occurring, you might lose data.

. On each controller, make sure the NV Caching Active (green) LED is off.
+
When the NV Caching Active (green) LED is off, any data in cache memory has been written to the drives and it is safe to continue with the procedure.
+
NOTE: If the NV Caching Active (green) LED is on, cached data is being written to the drives. You must wait for the process to complete and the NV Caching Active (green) LED to turn off. However, if the LED remains on for longer than five minutes, contact https://mysupport.netapp.com/site/global/dashboard[NetApp Support] before continuing with this procedure.
+
The NV Caching Active (green) LED is located next to the NV icon on the controller.

.. Select *Storage* > *Operations in progress* to see the Overview screen. 
.. Wait for all operations to complete and the Overview screen displays *No operations currently in progress*, before continuing to the next step.

== Step 3: Change the protocol

Change the protocol on the host I/O modules to a supported protocol, using SANtricity System Manager.

//Need to create includes
//syntax example:
// include::../_include/ge_io_module_upgrade_power_off_controllers.adoc[]

.Steps

. Select *Settings* > *System*.
. In the *Additional Settings* section, click *Change host I/O protocol* to open that page.
. In the *Host I/O Protocol* field, click the drop-down menu and choose the protocol you want to use.
. Type *CHANGE* in the open field to confirm your change. 
. Click the *Change* button.
